OpenClaw（前身为 Moltbot/Clawdbot）发展迅速。本指南涵盖了如何将现有安装安全迁移到最新版本 (v2026.1.30+)。

## 开始之前

:::caution
在进行重大升级之前，请务必备份您的数据。
:::

快照您当前的配置和工作区：
- **配置**: `~/.openclaw/openclaw.json` (或 `~/.moltbot/moltbot.json`)
- **凭据**: `~/.openclaw/credentials/`
- **工作区**: `~/.openclaw/workspace`

## 方法 1: 自动更新 (推荐)

最简单的更新方法是重新运行安装程序。它会检测现有的安装，进行就地升级，并自动运行 `openclaw doctor`。

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**注意:**
- 如果您不想再次运行入门向导，请添加 `--no-onboard`。
- 这适用于 **全局** 和 **源码** 安装。

## 方法 2: 全局安装更新

如果您通过 `npm` 或 `pnpm` 全局安装，可以使用包管理器进行更新。

**使用 npm:**
```bash
npm i -g openclaw@latest
```

**使用 pnpm:**
```bash
pnpm add -g openclaw@latest
```

更新后，运行 doctor 命令以确保一切迁移正确：

```bash
openclaw doctor
openclaw gateway restart
```

## 方法 3: 源码安装更新

如果您是从 git 克隆运行，请使用内置的更新命令：

```bash
openclaw update
```

此命令将：
1. 切换到选定的频道。
2. 获取并基于上游进行变基。
3. 重建依赖项和控制界面。
4. 运行 `openclaw doctor`。
5. 重启网关。

## 验证

更新后，验证您的安装状态：

```bash
openclaw --version
openclaw health
```

### v2026.1.30 新功能
- **每代理模型**: 使用 `openclaw gateway status --agent` 检查模型状态。
- **Shell 补全**: 运行 `openclaw completion` 设置 Zsh/Bash/PowerShell 的 Tab 补全。

## 故障排除

如果您遇到问题：
1. 运行 **`openclaw doctor`** - 它修复大多数常见的配置和迁移问题。
2. 检查日志：`openclaw logs --follow`。
3. 如有必要，您可以回滚到特定版本（请参阅 [更新文档](/zh/docs/install/updating)）。
