OpenClaw 可以在你已经使用的任何聊天应用上与你交谈。每个频道都通过网关（Gateway）连接。
任何地方都支持文本；媒体和表情反应的支持因频道而异。

## 支持的频道

- [WhatsApp](/docs/channels/whatsapp) — 最受欢迎；使用 Baileys 并且需要 QR 码配对。
- [Telegram](/docs/channels/telegram) — 通过 grammY 使用 Bot API；支持群组。
- [Discord](/docs/channels/discord) — Discord Bot API + Gateway；支持服务器、频道和私信（DM）。
- [Slack](/docs/channels/slack) — Bolt SDK；工作区应用。
- [Google Chat](/docs/channels/googlechat) — 通过 HTTP webhook 的 Google Chat API 应用。
- [Mattermost](/docs/channels/mattermost) — Bot API + WebSocket；频道、群组、私信（插件，需单独安装）。
- [Signal](/docs/channels/signal) — signal-cli；注重隐私。
- [BlueBubbles](/docs/channels/bluebubbles) — **推荐用于 iMessage**；使用 BlueBubbles macOS 服务器 REST API，支持全功能（编辑、撤回、特效、反应、群组管理 — 编辑功能目前在 macOS 26 Tahoe 上有问题）。
- [iMessage](/docs/channels/imessage) — 仅限 macOS；通过 imsg 进行原生集成（旧版，新设置请考虑 BlueBubbles）。
- [Microsoft Teams](/docs/channels/msteams) — Bot Framework；企业支持（插件，需单独安装）。
- [LINE](/docs/channels/line) — LINE Messaging API bot（插件，需单独安装）。
- [Nextcloud Talk](/docs/channels/nextcloud-talk) — 通过 Nextcloud Talk 的自托管聊天（插件，需单独安装）。
- [Matrix](/docs/channels/matrix) — Matrix 协议（插件，需单独安装）。
- [Nostr](/docs/channels/nostr) — 通过 NIP-04 的去中心化私信（插件，需单独安装）。
- [Tlon](/docs/channels/tlon) — 基于 Urbit 的信使（插件，需单独安装）。
- [Twitch](/docs/channels/twitch) — 通过 IRC 连接的 Twitch 聊天（插件，需单独安装）。
- [Zalo](/docs/channels/zalo) — Zalo Bot API；越南流行的信使（插件，需单独安装）。
- [Zalo Personal](/docs/channels/zalouser) — 通过 QR 登录的 Zalo 个人账户（插件，需单独安装）。
- [WebChat](/docs/web/webchat) — 通过 WebSocket 的 Gateway WebChat UI。

## 注意事项

- 频道可以同时运行；配置多个频道，OpenClaw 将根据聊天进行路由。
- 设置最快的通常是 **Telegram**（简单的 bot token）。WhatsApp 需要 QR 码配对，并且会在磁盘上存储更多状态。
- 群组行为因频道而异；请参阅 [群组](/docs/concepts/groups)。
- 为了安全起见，强制执行私信（DM）配对和白名单；请参阅 [安全性](/docs/gateway/security)。
- Telegram 内部机制：[grammY 说明](/docs/channels/grammy)。
- 故障排除：[频道故障排除](/docs/channels/troubleshooting)。
- 模型提供商有单独的文档；请参阅 [模型提供商](/docs/providers/models)。
