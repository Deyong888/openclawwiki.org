产品需求文档 (PRD): OpenClaw 部署与配置辅助工具套件
1. 项目概述 (Project Overview)
目标：在 Astro 框架搭建的文档网站（MoltbotWiki）中，集成一套交互式工具。 目的：
降低用户配置 OpenClaw/Moltbot/clawdbot 的技术门槛（解决 YAML 缩进、JSON 格式错误痛点）。
增加用户在页面的停留时间（Dwell Time）。
核心商业目标：在工具交互过程中，自然地根据用户的选择展示 VPS 联盟营销链接（Affiliate Links），提高转化率。

2. 技术栈 (Technical Stack)
主框架：Astro
UI 交互组件：React (或 Preact)
样式库：Tailwind CSS
图标库：Lucide React
其他：无需后端，纯前端逻辑。

3. 功能模块详细需求 (Feature Requirements)
功能一：Docker Compose 配置生成器 (The "No-Code" Config Generator)
优先级：P0 (最高/核心功能) 描述：用户通过表单选择部署环境和参数，右侧实时生成可直接复制的 docker-compose.yml 代码。

3.1 输入端 (Input Fields)
Deployment Platform (部署平台) - 单选框或下拉菜单
选项：Local (Mac/Windows), Vultr VPS, DigitalOcean VPS, Zeabur (Cloud),宝塔面板等。考虑后续可以扩展增加其他云平台。
交互逻辑：
选择 Vultr -> 显示带有“领取 $300 额度”的 Banner。
选择 DigitalOcean -> 显示带有“领取 $200 额度”的 Banner。
选择 Zeabur -> 隐藏代码框，显示“一键部署模板”按钮。
选择 Local -> 显示黄色警告：“本地运行需保持电脑常开，推荐使用云服务器。”
API Key Setup (API 设置)
输入框：Anthropic API Key (占位符 sk-ant-...，不实际存储，仅用于替换代码中的变量)。
Port Configuration (端口配置)
输入框：Host Port (默认 18789)。
Advanced Options (高级选项) - 折叠面板
开关：Enable Sandboxing (默认开启，挂载 /workspace)。
开关：Data Persistence (默认开启，挂载 /config)。

3.2 输出端 (Output Display)
代码预览区：实时显示生成的 YAML 内容。

功能按钮：
Copy to Clipboard (复制到剪贴板，点击后显示“Copied!”)。
Download .yml (下载文件)。

3.3 商业逻辑 (Affiliate Logic)
Vultr Banner 内容：
文案："Recommended for High Performance: Get $300 Free Credit on Vultr."
链接：https://www.vultr.com/?ref=9861424 (新窗口打开)。

样式：醒目的蓝色背景或边框。

功能二：Claude API 成本估算器 (Cost Estimator)
优先级：P1 描述：帮助用户预估运行 AI Agent 每月需要充值多少 API 余额。
拓展：考虑增加对其他模型（如 GPT-4、kimi2.5、minimax、deepseek、chatglm）的支持。
     增加对自定义指令的 Token 消耗计算。

3.1 输入端
Usage Intensity (使用强度)
滑动条：Hours per Day (范围 0-24)。
Task Complexity (任务复杂度)
单选：Low (Simple Chat), Medium (Coding), High (Heavy Research).
逻辑：对应不同的 Token 消耗速率（例如 Medium = 10k tokens/hour）。
Context Window (记忆长度)
开关：Enable Long Memory (开启后 Token 消耗 x2)。

3.2 输出端
Result Card (结果卡片)
显示：Estimated Daily Cost (预计日花费)。
显示：Estimated Monthly Cost (预计月花费)。
Comparison (对比文案)
文案："Still cheaper than hiring a real assistant ($2000+/mo)."

功能三：安全策略生成器 (Security Policy Maker)
优先级：P2 描述：生成 config.json 文件，定义 AI 的权限白名单。

3.1 输入端
Allowed Commands (允许的命令)
多选框组：git, npm, ls, cat, python, docker.
输入框：Custom Commands (允许用户补充)。
Work Directory (工作目录)
输入框：默认 /app/workspace。

3.2 输出端
代码预览区：显示 JSON 格式的配置。
功能按钮：Copy JSON。

4. 界面设计规范 (UI/UX Design)
布局：
采用 双栏布局 (Two-Column)：左侧为表单输入区，右侧为结果/代码预览区。
在移动端自动堆叠为单栏。

风格：
与 Astro Starlight 主题保持一致（深色模式 Dark Mode）。
代码块需要语法高亮 (Syntax Highlighting)。

交互：
输入时右侧代码实时更新，无延迟。
所有的 Affiliate Banner 必须设计得像“官方提示”或“优惠通知”，而不是烦人的弹窗广告。

功能四：为其他网站进行外链策略与植入规范

| 旧网站 | 外链位置 | 锚文本策略 |
|--------|---------|-----------|
| **stoiclines.store** | 选择最佳位置 | "philosophy blog", "digital stoicism" |
| **linktoai.top** | 工具推荐页/资源页 | "AI tools directory", "curated AI resources" |
| **24kwebgames.com** | 休闲推荐 | "online games", "browser games" |

外链植入规范

1. **自然植入**: 外链必须与内容相关，避免生硬插入
2. **多样化锚文本**: 使用品牌名、描述性短语、URL多种形式
3. **nofollow控制**: 重要页面使用dofollow，评论/UGC使用nofollow
4. **定期审计**: 每月检查外链有效性
